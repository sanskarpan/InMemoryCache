# Security Policy

## Supported versions

The `main` branch is the supported line for security fixes.

## Reporting a vulnerability

Do not open a public issue for a suspected security problem.

Report security concerns through the repository’s private security reporting channel if available, or contact the maintainers directly with:

- a short description of the issue
- the affected endpoint, package, or deployment path
- the impact and any known exploit path
- proof-of-concept steps if available

## What to include

Useful reports usually include:

- exact version or commit
- environment details
- request/response samples
- logs or stack traces
- whether the issue is reproducible in local Docker, Kubernetes, or production infrastructure

## Disclosure

We aim to acknowledge reports quickly, verify the impact, and release a fix before public disclosure whenever practical. Please keep reports confidential until a fix or coordinated disclosure plan exists.
